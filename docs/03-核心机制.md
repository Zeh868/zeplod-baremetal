# 03 核心机制

本章说明 **SRT 域**最常用的四个机制：事件、内存池、模块生命周期、数据通道。混合域组件见 [02-混合域与执行模型](02-混合域与执行模型.md)。

## 事件系统（`bm_event`）

发布/订阅模型，用于模块间异步通信。

### 基本流程

```c
/* 1. 注册事件类型（启动早期，单线程） */
bm_event_register_type(EVENT_TEMP, "TEMP");

/* 2. 订阅 */
bm_event_subscriber_id_t sub_id;
bm_event_subscribe(EVENT_TEMP, on_temp, user_ctx, &sub_id);

/* 3. 发布（推荐拷贝语义，避免栈指针悬空） */
sensor_data_t data = { .temp = 250 };
bm_event_publish_copy(EVENT_TEMP, BM_EVENT_PRIO_NORMAL, &data, sizeof(data));

/* 4. 主循环处理 */
bm_event_process();
```

### 要点

| 话题 | 说明 |
|------|------|
| 优先级 | 数值越小优先级越高；出队时高优先级先处理 |
| 队列深度 | `BM_CONFIG_EVENT_QUEUE_SIZE`，满则丢弃并可通过 `bm_event_get_dropped_count()` 统计 |
| ISR 发布 | 仅 `bm_event_publish_copy` / `bm_event_publish_event`；订阅回调仍在 SRT 执行 |
| 零拷贝 | `publish_event` 不拷贝 payload，调用方须保证生命周期到处理完成 |
| 测试复位 | `bm_event_reset()` 在 `setUp` 或启动时建立确定初始状态 |

### 与 Ultra 的差异

编译期绑定 → 运行时注册/订阅；支持多订阅者与优先级。对照表见 [08-迁移与演进](08-迁移与演进.md#ultra--core)。

## 内存池（`bm_mempool`）

固定大小对象池，避免堆分配。

```c
BM_MEMPOOL_DEFINE(sensor_pool, sensor_data_t, 4);

sensor_data_t *obj = bm_mempool_alloc(&sensor_pool);
/* 使用 obj */
bm_mempool_free(&sensor_pool, obj);
```

- 池与对象在编译期静态分配
- 双释放会被拒绝（fail-stop，返回错误）
- 与事件配合：发布方 `publish_copy` 后通常立即 `free`，订阅方只读副本

## 模块生命周期（`bm_module`）

用一张表描述 init → start → stop → deinit 顺序：

```c
const bm_module_t _bm_module_table[] = {
    { .name = "sensor", .priority = 0, .init = sensor_init, .start = sensor_start },
    { .name = "comm",   .priority = 1, .init = comm_init,   .start = comm_start },
};
const uint32_t _bm_module_count = 2;

int main(void) {
    bm_event_reset();
    bm_module_init_all();   /* 按 priority 升序 */
    bm_module_start_all();
    for (;;) { bm_event_process(); }
}
```

- `priority` 越小越先初始化
- `init` / `start` 任一步失败会回滚已成功的模块
- 表项过多会返回 `BM_ERR_OVERFLOW`（有界表）

## 数据通道（`bm_channel`）

单生产者单消费者（SPSC）环形缓冲，**仅在 SRT 域使用**（禁止 HRT ISR 访问）。

```c
BM_CHANNEL_DEFINE(sensor_ch, sensor_sample_t, 8);

sensor_sample_t sample = { ... };
bm_channel_send(&sensor_ch, &sample);

sensor_sample_t out;
if (bm_channel_recv(&sensor_ch, &out) == BM_OK) { /* 处理 */ }
```

- `depth` 编译期须 ≥ 2
- 满时 `send` 返回 `BM_ERR_OVERFLOW`；空时 `recv` 返回 `BM_ERR_WOULD_BLOCK`
- 索引损坏时 fail-stop 返回 `BM_ERR_INVALID`

## 软件看门狗（`bm_wdg`）

多逻辑模块分别注册，全部在 `BM_CONFIG_WDG_MODULE_TIMEOUT_MS` 内 `feed_module` 后才允许喂硬件狗：

```c
bm_wdg_register("comm");
bm_wdg_register("control");

/* 各模块周期调用 */
bm_wdg_feed_module("comm");

/* 主循环 */
bm_wdg_feed();
```

## 临界区（`bm_critical`）

`BM_CRITICAL_ENTER()` / `BM_CRITICAL_EXIT()` 由 HAL 实现，屏蔽与 SRT 共享数据相关的可抢占中断。HRT 路径不应长时间持锁。

## 日志（`bm_log`）

分级宏 `BM_LOGI` / `BM_LOGW` / `BM_LOGE` 等，输出经 `bm_hal_uart` 或平台重定向。**禁止在 HRT ISR 中使用。**
