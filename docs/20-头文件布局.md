# 20 头文件布局

> **本文职责**：说明 `include/` 目录分层与各类角色应包含哪些头文件。  
> **CMake 如何注入路径** → [05-构建配置与CMake](05-构建配置与CMake.md)。

## 1. 目录结构

应用代码仍使用扁平 `#include "bm_event.h"`；物理目录按职责分层：

```text
include/
├── bm/
│   ├── common/     # 全框架共用：types、log、config、atomic…
│   ├── core/       # 核心域：event、mempool、module、shell、wdg…
│   ├── hybrid/     # 混合域：hrt、ticker、ctrl_inst、sync…
│   ├── hal/        # 应用 HAL 契约（bm_hal_*）
│   └── ultra/      # 极裁剪头文件库
└── drv/            # 驱动 API（bm_drv_*），后端实现者使用
```

## 2. 谁该包含哪一层

| 角色 | 典型头文件 | 目录 |
|------|-----------|------|
| 应用 | `bm_event.h`、`bm_hal_pwm.h`、`bm_hrt.h` | `bm/common` `bm/core` `bm/hybrid` `bm/hal` |
| Ultra 固件 | `bm_ultra.h` | `bm/ultra`（+ `bm/common` 若需要 types） |
| 平台 Port | `bm_drv_pwm.h`、`bm_drv_timer.h` | 上述全部 + `drv/` |

## 3. CMake

`cmake/bm_include_dirs.cmake` 定义 `BM_INCLUDE_APP_DIRS` 与 `BM_INCLUDE_BACKEND_DIRS`。  
`bm_config` 目标导出后端路径（`bm_hal` 分发层依赖 `bm_drv_*` 类型）。

## 4. 手动 IDE 配置

Keil / IAR 请在 Include Paths 中加入 `include/bm/common`、`core`、`hybrid`、`hal`、`ultra`；编写 Port 时再加 `include/drv`。详见 [18-Keil集成](18-Keil集成.md)、[19-IAR集成](19-IAR集成.md)。
