# 20 头文件布局

> **本文职责**：说明 `include/` 公共 API 布局与统一入口。  
> **CMake** → [05-构建配置与CMake](05-构建配置与CMake.md)。

## 1. 原则

**所有对外头文件均在 `include/` 根目录**，无子目录、无转发 shim。集成时 Include Path 只需一条：`zeplod-baremetal/include/`。

## 2. 统一入口

```c
#include "zeplod.h"
```

`zeplod.h` 读取 `bm_config.h` 中的 `BM_CONFIG_ENABLE_*`，按需 `#include` 各子系统头文件。`bm_hal_*` 不聚合，由应用按板卡单独包含。

CMake 集成时，`bm_config` 目标将 `BM_ENABLE_*` 同步为 `BM_CONFIG_ENABLE_*` 编译宏。

### 组件开关（`bm_config.h`）

| 宏 | 默认 | CMake 选项 |
|----|------|------------|
| `BM_CONFIG_ENABLE_ULTRA` | 0 | — |
| `BM_CONFIG_ENABLE_MODULE` | 1 | `BM_ENABLE_MODULE` |
| `BM_CONFIG_ENABLE_CHANNEL` | 0 | `BM_ENABLE_CHANNEL` |
| `BM_CONFIG_ENABLE_SHELL` | 0 | `BM_ENABLE_SHELL` |
| `BM_CONFIG_ENABLE_WDG` | 1 | `BM_ENABLE_WDG` |
| `BM_CONFIG_ENABLE_HRT` | 0 | `BM_ENABLE_HRT` |
| `BM_CONFIG_ENABLE_TICKER` | 0 | `BM_ENABLE_TICKER` |
| `BM_CONFIG_ENABLE_CTRL_INST` | 0 | `BM_ENABLE_CTRL_INST` |
| `BM_CONFIG_ENABLE_SYNC` | 0 | `BM_ENABLE_SYNC` |

依赖链：`SYNC → CTRL_INST → HRT`。

### 分层聚合头（可选）

| 头文件 | 用途 |
|--------|------|
| `zeplod.h` | **主入口** |
| `bm_lite.h` | Core + 可选 module/channel/shell/wdg |
| `bm_core.h` | 仅事件、内存池、类型 |
| `bm_hybrid.h` | HRT 混合域 |
| `bm_ultra.h` | Ultra 头文件库 |

## 3. 目录结构

```text
include/
├── zeplod.h           # 统一对外 API 入口
├── bm_config.h        # 容量与组件开关
├── bm_event.h …       # 框架子系统
├── bm_hal_*.h         # 应用 HAL 契约
├── bm_drv_*.h         # Port 驱动 API
└── README.md
```

## 4. 角色与典型头文件

| 角色 | 推荐 |
|------|------|
| 应用 | `zeplod.h` + 按需 `bm_hal_*` |
| Ultra | `zeplod.h`（`BM_CONFIG_ENABLE_ULTRA=1`）或 `bm_ultra.h` |
| Port | `bm_drv_*.h` + `bm_hal_*.h` 实现 |

## 5. CMake / IDE

`BM_INCLUDE_APP_DIRS` = `include/`。Keil / IAR 见 [18](18-Keil集成.md)、[19](19-IAR集成.md)。  
`tools/list_sources.py --list-includes` 输出同一路径。
