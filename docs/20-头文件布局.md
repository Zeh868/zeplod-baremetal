# 20 头文件布局

> **本文职责**：说明 `include/` 对外 API 与子目录分类。  
> **CMake** → [05-构建配置与CMake](05-构建配置与CMake.md)。

## 1. 原则

- **对外 API** 在 `include/` **根目录**（入口少、清晰）
- **实现头文件** 按职责分子目录；应用经 `zeplod.h` 间接包含，无需记全路径
- 应用 Include Path：**一条** `zeplod-baremetal/include/`
- 编译库源码 / Port 时 CMake 额外注入子目录（`bm_target_internal_includes`）

## 2. 根目录（对外）

| 头文件 | 用途 |
|--------|------|
| `zeplod.h` | **主入口** |
| `bm_config.h` | 默认配置模板 |
| `bm_core.h` | 事件 + 内存池 + 类型 |
| `bm_lite.h` | Core + 可选 module/channel/shell/wdg |
| `bm_hybrid.h` | HRT / ticker / exec / sync |
| `bm_ultra.h` | Ultra 剖面 |
| `bm_hal.h` | HAL 核心外设聚合（可选） |

```c
#include "zeplod.h"
#include "hal/bm_hal_uart.h"
```

`BM_CONFIG_ENABLE_*` 与 CMake `BM_ENABLE_*` 对齐，见 `bm_config.h`。

## 3. 子目录

```text
include/
├── zeplod.h  bm_config.h  bm_*.h（聚合）  bm_hal.h
├── bm/
│   ├── common/     types、log、atomic、time…
│   ├── core/       event、mempool、module、shell、wdg、channel
│   └── hybrid/     hrt、ticker、exec、sync、snapshot…
├── hal/            bm_hal_*.h
└── drv/            bm_drv_*.h（Port）
```

## 4. 角色

| 角色 | 推荐 |
|------|------|
| 应用 | `zeplod.h` + `hal/bm_hal_*.h` |
| Ultra | `BM_CONFIG_ENABLE_ULTRA=1` |
| Port | `drv/bm_drv_*.h` + `zeplod_port_include_dirs()` |

## 5. CMake / IDE

- 应用：`BM_INCLUDE_PUBLIC_DIRS` = `include/`
- 库与 Port：`BM_INCLUDE_INTERNAL_DIRS` 含 `bm/*`、`hal/`、`drv/`
- Keil：`python tools/list_sources.py --list-includes`（应用一条 + Port 多条）
