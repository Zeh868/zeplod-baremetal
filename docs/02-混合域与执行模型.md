# 02 混合域与执行模型

混合域（Hybrid Domain）解决一个问题：**电流环、PWM 同步采样**不能和 **Shell、通信状态机** 挤在同一条软件路径里。框架把执行分为三类，并强制隔离共享数据结构。

## 三类执行路径

| 执行类 | 典型频率 | 触发方式 | 适用任务 |
|--------|----------|----------|----------|
| **Hardware HRT** | 5–200 kHz | PWM 更新、ADC 完成、比较器 | 电流环、数字电源、硬件保护 |
| **Scheduled HRT** | 100 Hz–10 kHz | 专用定时器 ISR（`bm_hrt`） | 速度环、低频采样 |
| **SRT** | 无硬截止 | 主循环 `bm_event_process` / `bm_ticker_poll` | 状态机、通信、Shell、诊断 |

## 核心边界规则

1. **HRT ISR 禁止访问** `bm_event` 队列、`bm_mempool`、`bm_channel` 及任何由 SRT 临界区保护的结构。
2. **亚微秒级保护**靠比较器、定时器 Break 等硬件完成，不依赖 CPU 调度。
3. **跨域传数据**使用 `bm_snapshot`（三缓冲邮箱）或应用侧无锁队列，不用事件队列“捎带”实时数据。
4. **Hardware HRT 优先级高于 Scheduled HRT**；分析 WCET 时必须计入抢占。

```text
PWM/ADC/COMP IRQ                主循环
       |                            |
       v                            v
 Hardware HRT 回调            bm_event_publish_copy()
       |                            |
       +---- bm_snapshot / 邮箱 -----+
       |
 Scheduled HRT（bm_hrt 定时 ISR）
       |
       v
  有界槽位分发（每 ISR 每槽最多 1 次）
```

## Hardware HRT：HAL 绑定

由 HAL 在中断里直接调用已注册的回调，不经过 `bm_hrt` 扫描：

```c
bm_hal_hrt_binding_t binding = {
    .callback = current_loop_isr,
    .context  = &motor_state,
};
bm_hal_pwm_bind_update(&pwm, &binding);
```

HAL ISR 只做：清标志 → 取 ADC 结果 → 调回调 → 返回。**禁止**在 ISR 里 `bm_event_publish`、`BM_LOG`、遍历全局表。

解绑时：先关中断源，再清回调（`binding == NULL`）。

## Scheduled HRT：`bm_hrt`

用于不需要与 PWM 相位锁定的中低频硬实时任务：

```c
static const bm_hrt_slot_t slots[] = {
    {
        .period_us = 1000,
        .trigger   = BM_HRT_TRIGGER_TIMER,
        .callback  = speed_loop,
        .context   = &axis,
        .name      = "speed",
    },
};
bm_hrt_init(slots, 1);
bm_hrt_start();
```

配置项 `BM_CONFIG_HRT_TICK_US` 决定时间粒度；槽位数受 `BM_CONFIG_HRT_MAX_SLOTS` 限制。

## SRT：`bm_ticker` 与事件

- **`bm_ticker`**：在主循环调用 `bm_ticker_poll()`，按毫秒周期触发软实时任务（通信、慢环）。
- **`bm_event`**：异步消息、模块间解耦；订阅者在 `bm_event_process()` 中按优先级出队。

二者都在 SRT 域，可安全使用临界区与 `bm_channel`。

## 多实例控制：`bm_ctrl_inst`

一个产品可有多个控制算法实例（多轴、多电芯）：

```c
BM_CTRL_INST_DEFINE(axis0, ...);
BM_CTRL_INST_DEFINE(axis1, ...);
bm_ctrl_init_all();   // 校验资源与绑定
bm_ctrl_start_all();  // 失败则回滚并停 HRT
```

每个实例声明占用的 PWM/ADC/定时器等资源；`bm_resource` 在启动前做冲突检测。

## 同步域：`bm_sync`

多路 PWM/ADC 需要固定相位关系时（交错电源、多关节同步），使用同步域将多个实例绑定到同一相位基准。

## 跨域数据：`bm_snapshot`

HRT 写入最新一帧测量值，SRT 读取稳定快照，避免读写竞态：

```text
HRT: bm_snapshot_write()  →  三缓冲轮换
SRT: bm_snapshot_read()   →  只读已完成帧
```

## 主循环模板（Control 级）

```c
void main_loop(void) {
    for (;;) {
        bm_ticker_poll();
        bm_event_process();
        bm_shell_poll(&shell);   /* 仅调试构建 */
        bm_wdg_feed();           /* 各模块需先 feed_module */
    }
}
```

HRT 路径在 ISR 中独立运行，不进入上述循环。
