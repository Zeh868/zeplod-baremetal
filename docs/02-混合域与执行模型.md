# 02 混合域与执行模型

> **本文职责**：当前三类执行路径（Hardware / Scheduled / SRT）与域边界摘要。
> **不负责**：ISR 接线细节 → [03](03-执行域与跨域通讯.md)。

电流环、PWM 同步采样不能与 Shell、通信挤在同一条软件路径。**混合域**按时间约束把执行分为三类，并规定跨域只能走专用通道。

## 三类执行路径

| 执行类 | 典型频率 | 谁触发 | 框架入口 |
|--------|----------|--------|----------|
| **Hardware HRT** | 5–200 kHz | ADC/PWM/COMP 硬件 IRQ | HAL `bind_*` 保存 callback，向量 ISR 调用 |
| **Scheduled HRT** | 100 Hz–10 kHz | 定时器 tick ISR | `bm_hrt_init` + `bm_hrt_start` 扫描槽表 |
| **SRT** | 无硬截止 | 主循环 | `bm_event_process` / `bm_ticker_poll` |

**要点**：Hardware HRT **不经过** `bm_hrt`。`bm_hrt` 只分发 `BM_HRT_TRIGGER_TIMER` 槽。术语对照见 [03 §1](03-执行域与跨域通讯.md#1-术语对照)。

## 规划中的 Block/Frame RT

音频、FFT、高速采集和视觉不是逐样本控制，也不是无截止期 SRT。多领域路线
新增第四类执行路径：

| 执行类 | 触发 | 数据粒度 | deadline |
|--------|------|----------|----------|
| **Block/Frame RT（规划）** | DMA half/full、line/frame ready | PCM/ADC block、图像行/帧 | 下一个块/帧到来前 |

该域使用静态零拷贝 `bm_stream` 和有界 `bm_pipeline`，不使用事件队列传输
连续 payload。详细设计见 [23-多领域确定性流式架构](23-多领域确定性流式架构.md)。
在这些 API 实现前，本文其余规则只描述当前 Control 级运行时。

## 域边界（摘要）

| HRT ISR 禁止 | HRT ISR 允许 | 跨域数据 | 跨域信号 |
|--------------|--------------|----------|----------|
| `bm_event_process`、常规池分配、`bm_channel`、`BM_LOG` | 读 ADC、写 PWM、`bm_snapshot`、`*_from_isr` 入队 | `bm_snapshot` | `bm_event_publish_copy_from_isr` |

完整规则与 WCET 注意：[03 §2](03-执行域与跨域通讯.md#2-为什么要分域)。跨域选型矩阵：[03 §7](03-执行域与跨域通讯.md#7-跨域通讯)。

## 运行时结构（Control 级）

```text
                    主循环 (SRT)
                    bm_ticker_poll → bm_event_process
                           ▲
                           │ snapshot / 事件（信号）
                           │
    ADC/PWM IRQ ──► Hardware HRT callback
                           │
    TIM tick ISR ─► bm_hrt → Scheduled HRT callback

bm_exec：描述多实例 + 槽位，init_all 时组装 HRT 表并 bind
bm_sync：多轴硬件相位对齐（不传测量值）
```

## Control 级启动顺序（摘要）

```text
bm_exec_init_all  →  bm_exec_start_all  →  （内部在全部 ops->start 后再 bm_hrt_start）
（可选 sync: configure → arm → trigger；trigger 为 one-shot，须重新 arm）
（可选 ticker: 在 subscribe 与 start_all 之后 ticker_init）
```

独立使用 `bm_hrt`（不经 `bm_exec`）时仍须手动 `bm_hrt_init` + `bm_hrt_start`。含 `bm_sync` 的完整顺序与常见错误：[07 §5.2](07-应用骨架与数据流.md#52-正确初始化顺序)。
