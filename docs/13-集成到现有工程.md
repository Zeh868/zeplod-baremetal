# 13 集成到现有工程

面向 **已有 CubeMX / MCUXpresso / Keil / IAR / CMake Demo**，以最小步骤把 Zeplod 当作库接入。

详细模板与厂商说明见仓库 [`integration/`](../integration/README.md)。

## 核心原则

| 原则 | 说明 |
|------|------|
| 不替换厂商启动代码 | 保留 startup、`SystemInit`、链接脚本 |
| 配置一处 | 应用目录下的 `bm_config.h` 必须在 include 路径最前 |
| 后端可插拔 | `register_*` 参考实现或 `external` 胶水对接 `HAL_*` / SDK |
| 按裁剪链接 | 用 `PROFILE` 或 `list_sources.py`，避免把整个 `src/` 塞进工程 |

## CMake：一行配置 + 一行链接

```cmake
include(.../zeplod-baremetal/cmake/zeplod.cmake)
zeplod_configure(ROOT ... PROFILE event BACKEND register_stm32g4 CONFIG .../bm_config.h)
zeplod_link(your_cubemx_target)
```

| PROFILE | 典型用途 |
|---------|----------|
| `minimal` | 仅事件/内存池/日志 |
| `event` | 事件 + 模块 + 看门狗（多数业务 Demo） |
| `servo` | HRT + ticker + ctrl_inst（伺服/BMS 电流环） |
| `full` | event + servo 组件全开 |

| BACKEND | 说明 |
|---------|------|
| `register_stm32g4` 等 | 仓库内寄存器参考，快速冒烟 |
| `external` | 自研胶水，对接厂商 HAL（**量产推荐**） |
| `native_sim` | 仅 PC 单元测试 |

## 非 CMake IDE

```bash
python tools/list_sources.py --profile event --backend register_stm32g4 --format keil
python tools/list_sources.py --profile event --backend register_stm32g4 --list-includes --format iar
```

定义 `ZEPLOD_ROOT` 宏指向框架根，将输出路径加入工程。

## 与 08 HAL 移植的关系

- **13（本文）** — 怎么把库 **挂进** 现有工程
- **08** — 后端 **实现什么**、driver API 契约

量产路径：`BACKEND external` + 阅读 `platform/backends/register_<mcu>/` 作模板 → 在胶水层调用 Cube/SDK API。

## 检查清单

- [ ] `bm_config.h` 在应用 include 路径首位
- [ ] 已链 HAL 分发层（`src/hal/*.c`）与后端或胶水
- [ ] 未把 `native_sim` / `qemu` 后端链入真机
- [ ] `BM_CONFIG_EVENT_QUEUE_SIZE` 为 2 的幂且 ≥ 2
- [ ] 混合域已在向量 ISR 中调用 HAL 绑定回调（见 03）
