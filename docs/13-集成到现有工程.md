# 13 集成到现有工程

Zeplod 作为 **库** 接入已有 Cube / MCUXpresso / Keil / IAR 工程。  
完整说明：[integration/README.md](../integration/README.md)。

## 两件事，顺序固定

| 步骤 | 做什么 | 类比 FreeRTOS |
|------|--------|----------------|
| **① 移植** | 写 `bm_port.c`，实现 `bm_drv_*_api`，接厂商 HAL | `portable/.../port.c` |
| **② 集成** | 源码加入工程，或链接 `libbm_*.a` | 内核源码 / `libfreertos.a` |

配置头文件：`bm_config.h`（类比 `FreeRTOSConfig.h`），须位于应用 include 路径 **最前**。

## 集成方式二选一

### 方式 A — 源码集成

库 `.c` 与 Port `.c` 都作为源文件加入厂商 IDE 工程。

```bash
python tools/list_sources.py --profile event --format keil          # 库源文件
python tools/list_sources.py --profile event --list-includes --format keil
```

另加：`integration/port/bm_port.c`（移植层，**不在** `list_sources` 输出里）。

### 方式 B — 静态库集成

```bash
cmake -B build -S integration/static-lib \
    -DCMAKE_TOOLCHAIN_FILE=cmake/toolchain-arm-none-eabi.cmake \
    -DZEPLOD_PROFILE=event
cmake --build build
```

链接 `libbm_core.a`、`libbm_hal.a` 等 + 头文件；**仍须**在应用工程编译 `bm_port.c`。

## CMake 工程（两种方式都适用）

```cmake
include(.../cmake/zeplod.cmake)
zeplod_configure(ROOT ... PROFILE event BACKEND external CONFIG .../bm_config.h)
target_sources(my_app PRIVATE Core/Src/bm_port.c)
zeplod_link(my_app)
```

- `zeplod_configure` + `zeplod_link`：自动以 **源码** 方式拉入库目标（方式 A′）。
- 若已用方式 B 链好 `.a`，只需 `target_sources` 加 Port，并自行 `target_include_directories`。

## PROFILE

| 值 | 用途 |
|----|------|
| `minimal` | 仅核心事件 |
| `event` | 事件 + 模块 + 看门狗 |
| `servo` | 混合域控制 |
| `full` | 全开 |
