# 01 框架概览与资源层级

> **本文职责**：框架定位、组件地图、资源层级与 CMake 目标总览。  
> **不负责**：构建选项细节 → [05](05-构建配置与CMake.md)；挂库步骤 → [13](13-集成到现有工程.md)。

## 定位

Zeplod Baremetal 是面向**机电控制节点**的裸机事件驱动框架：电机驱动、数字电源、BMS、机器人关节等。在 RTOS 过重或不可用的场景，提供从 8 位 MCU 到 Cortex-M 的统一编程模型，并可向 [Zeplod on Zephyr](https://github.com/zeplod/zeplod) 演进。

**不是**通用 RTOS 替代品，而是**有界、零堆、可裁剪**的控制专用运行时。

## 设计原则（使用时要记住）

1. **静态优先**：表、队列、池在编译期或早期 `init()` 分配，不用 `malloc`。
2. **显式契约**：资源声明、HRT 槽位、跨域数据路径必须写清楚。
3. **有界执行**：所有循环与队列有已知上限；HRT 回调禁止阻塞、分配、打日志。
4. **故障安全**：初始化先校验再动硬件；失败走确定性回滚（fail-stop），见 [11-安全与可靠性](11-安全与可靠性.md)。

## 组件地图

```text
应用
 ├── bm_module / bm_channel / bm_shell / bm_wdg  （可选）
 │        └── bm_core
 ├── bm_hrt / bm_ticker / bm_ctrl_inst / bm_sync / bm_snapshot  （混合域，可选）
 │        └── bm_core + HAL
 └── bm_ultra.h  （极裁剪，无 src 链接）
```

| 组件 | 作用 | 默认 CMake |
|------|------|------------|
| `bm_core` | 事件、内存池、临界区 | 必选 |
| `bm_module` | 模块 init/start/stop 表驱动 | ON |
| `bm_channel` | SPSC 环形通道（仅 SRT） | OFF |
| `bm_shell` | 串口命令行（调试用） | OFF |
| `bm_wdg` | 软件看门狗聚合喂硬件狗 | ON |
| `bm_hrt` | Scheduled HRT 定时分发 | OFF |
| `bm_ticker` | SRT 周期任务（主循环） | OFF |
| `bm_ctrl_inst` | 多实例控制 + 资源声明 | OFF |
| `bm_sync` | 多轴相位同步域 | OFF |

依赖关系：`bm_sync` → `bm_ctrl_inst` → `bm_hrt`。详见 [05-构建配置与CMake](05-构建配置与CMake.md)。

## 资源层级

按 Flash/RAM 与能力选择组件子集：

| 层级 | Flash | RAM | 典型 MCU | 组件 |
|------|-------|-----|----------|------|
| **Ultra** | < 8 KB | < 1 KB | STM8、AVR | `bm_ultra.h` 头文件库 |
| **Nano** | 8–32 KB | 1–4 KB | CH32V003、STM32F030 | `bm_core` + 可选 `bm_module` |
| **Lite** | 32–128 KB | 4–16 KB | STM32F103 | + `bm_channel`、`bm_shell` |
| **Control** | 32 KB+ | 4 KB+ | STM32G4、STM32F3 | + `bm_hrt`、`bm_ctrl_inst`、`bm_sync` |

升级路径见 [10-迁移与演进](10-迁移与演进.md)。

## 依赖方向

```text
application  (#include "zeplod.h")
  → 可选组件（module/channel/shell/wdg/hrt/ctrl，由 BM_CONFIG_ENABLE_* 裁剪）
    → bm_core
      → HAL 契约（include/bm_hal_*.h，按需单独包含）
```

- 统一入口：`zeplod.h`（推荐）；分层伞头：`bm_lite.h`、`bm_hybrid.h`、`bm_core.h`
- 公共类型与错误码：`bm_types.h`
- 事件 API：`bm_event.h`；内存池：`bm_mempool.h`；临界区原子：`bm_atomic.h`
- 组件头文件只依赖 `bm_types.h`，不反向依赖 HAL；HAL 头文件也不反向依赖框架伞头

## 源码布局

```text
Source/core/    SRT 框架与可选组件（module、channel、shell、wdg）
Source/hybrid/  HRT、ticker、多实例控制、sync 域
Source/hal/     弱符号 HAL 分发，由 portable/ Port 覆盖
portable/       Port 模板与参考后端（见 08、14）
```

公共头文件均在 `include/` 根目录，见 [20-头文件布局](20-头文件布局.md)。

## CMake 目标

| 目标 | 说明 |
|------|------|
| `bm_core` | 必选：事件、内存池、临界区 |
| `bm_module`、`bm_channel`、`bm_shell`、`bm_wdg` | 可选组件 |
| `bm_framework` | 聚合当前已启用组件的接口目标 |
| `bm_config` | 各框架目标共享的编译设置 |

应用应只链接用到的组件；若产品刻意全开可链 `bm_framework`。  
通过 `BM_CONFIG_FILE` 或仓库根 `bm_config.h` 强制包含配置宏。详见 [05-构建配置与CMake](05-构建配置与CMake.md)。

## 状态所有权

每个子系统在单 MCU 链接映像内各有一份静态运行时（事件总线、HRT、ticker、ctrl 批处理会话、sync 域）。多轴控制使用多个 `bm_ctrl_inst_t`，而非多个事件总线。只读查询 API（`bm_ctrl_get_session`、`bm_hrt_is_started` 等）供测试与监督使用。详见 [12-运行时与实例模型](12-运行时与实例模型.md) 与 `include/bm_runtime_model.h`。

应用启动与测试 `setUp()` 中应调用 `bm_event_reset()` 以建立确定性初始状态。

## 典型产品形态

| 场景 | 推荐组合 |
|------|----------|
| 传感器节点 + 状态机 | Nano：`bm_core` + `bm_module` + `bm_event` |
| 带调试串口的多任务 | Lite：+ `bm_channel`、`bm_shell` |
| 单轴伺服 | Control：Hardware HRT（PWM/ADC）+ Scheduled HRT + SRT 状态机 |
| 多电芯 BMS | Control：多 `bm_ctrl_inst` + `bm_snapshot` 上报 SRT |
