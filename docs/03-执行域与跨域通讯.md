# 03 执行域与跨域通讯

本文是 [02-混合域与执行模型](02-混合域与执行模型.md) 的**原理深化**：说明三类执行路径在代码里如何分叉、同步域解决什么问题、跨域该用什么机制。建议先读 `02` 的概念表，再读本文。

---

## 1. 术语对照

文档与代码里「HRT」一词有两层含义，容易混淆：

| 叫法 | 代码标识 | 触发源 | 执行上下文 |
|------|----------|--------|------------|
| **Hardware HRT**（常简称 HSRT） | `BM_CTRL_SLOT_HARDWARE` | PWM 更新、ADC 完成、比较器 IRQ | HAL 在中断里**直接**调回调 |
| **Scheduled HRT** | `BM_CTRL_SLOT_SCHEDULED` → `bm_hrt` | 专用定时器 tick（如 TIM6） | `bm_hrt` ISR 扫描到期槽 |
| **SRT** | 主循环 | 无硬件截止 | `bm_event_process` / `bm_ticker_poll` |

**关键实现事实**（读源码时务必记住）：

- `bm_hrt` **只分发** `BM_HRT_TRIGGER_TIMER` 的槽；`bm_hrt.c` 中 `hrt_dispatch()` 会跳过其他触发类型。
- **Hardware HRT 不经过 `bm_hrt`**。`bm_ctrl_init_all()` 通过 `bind_hardware()` 把 HAL IRQ 绑到 `bm_ctrl_run_binding()`，再由它调用你的 `step` 函数。
- `bm_hrt_trigger_t` 枚举里的 `PWM_UPDATE` / `ADC_COMPLETE` 用于 **slot 描述**；Hardware 路径由 HAL 绑定实现，Scheduled 路径只用 `TIMER`。

```text
                    ┌──────────────────────────────────────┐
  ADC/PWM IRQ ─────►│ Hardware HRT                         │
                    │  HAL → bm_ctrl_run_binding → step    │
                    └──────────────────────────────────────┘

  TIM6 tick   ─────►│ Scheduled HRT (bm_hrt)               │
                    │  hrt_dispatch → callback → step      │
                    └──────────────────────────────────────┘

  main loop   ─────►│ SRT                                  │
                    │  bm_ticker_poll / bm_event_process   │
                    └──────────────────────────────────────┘
```

## 2. 为什么要分域

裸机机电控制里同时存在两类矛盾需求：

1. **硬截止任务**：电流环必须在 PWM 周期内完成，WCET 以微秒计。
2. **软任务**：通信协议、Shell、显示、复杂状态机，允许毫秒级抖动。

若共用同一条执行路径和同一套数据结构（事件队列、内存池、日志），会出现：

- HRT ISR 里 `malloc` / 遍历订阅表 → WCET 不可分析；
- SRT 持锁时被高优先级 HRT 抢占 → 数据结构不一致；
- 用事件队列传递 20 kHz 采样流 → 队列爆满、延迟不可控。

框架的解法是 **静态分区**：按时间约束划分执行域，并规定跨域只能通过专用通道通讯。

### 2.1 域边界规则

| 规则 | 说明 |
|------|------|
| HRT ISR 禁止 | `bm_event_process`、订阅回调、`bm_mempool_alloc`（常规路径）、`bm_channel`、`BM_LOG` |
| HRT ISR 允许 | 读 ADC、写 PWM/比较器、写 `bm_snapshot`、调 `bm_event_publish_copy_from_isr`（仅入队，不执行回调） |
| SRT 允许 | 事件、池、通道、Shell、看门狗、读 `bm_snapshot` |
| 亚微秒保护 | 比较器、定时器 Break 等硬件路径，不依赖 CPU 调度 |

`bm_critical` 保护的是 **SRT 共享结构**。HRT 路径不应长时间持锁；分析 WCET 时须计入 Hardware HRT 对 Scheduled HRT 的抢占。

## 3. Hardware HRT 用法

### 3.1 直接 HAL 绑定（不用 `bm_ctrl_inst`）

适合单环、资源极简的场景：

```c
static void current_loop_isr(void *ctx) {
    motor_state_t *m = (motor_state_t *)ctx;
    uint16_t adc;
    bm_hal_adc_read_injected(&adc_periph, 0, &adc);
    m->duty = pi_update(&m->pi, adc);
    bm_hal_pwm_set_duty(&pwm, 0, m->duty);
}

bm_hal_hrt_binding_t binding = {
    .callback = current_loop_isr,
    .context  = &motor_state,
};
bm_hal_adc_bind_complete(&adc_periph, &binding);
```

解绑：先关中断源，再 `bind_*(..., NULL)`。

### 3.2 通过 `bm_ctrl_inst`（推荐，多实例）

在实例的 `slots[]` 中声明 `BM_CTRL_SLOT_HARDWARE`：

```c
static const bm_ctrl_slot_t g_servo_slots[] = {
    {
        BM_CTRL_SLOT_HARDWARE,
        0u,                              /* period_us 对硬件槽无效 */
        BM_HRT_TRIGGER_ADC_COMPLETE,
        current_step,                    /* 你的环算法 */
        bind_adc,                        /* 绑 HAL */
        "current"
    },
};
```

`bind_adc` 典型实现：

```c
static int bind_adc(const bm_ctrl_inst_t *inst,
                    const bm_hal_hrt_binding_t *binding) {
    (void)inst;
    return bm_hal_adc_bind_complete(&BM_HAL_ADC0, binding);
}
```

`bm_ctrl_init_all()` 会为每个 Hardware 槽构造 `bm_hal_hrt_binding_t`，回调统一为 `bm_ctrl_run_binding`，最终调用 `current_step(instance)`。

**ISR 内应完成**：清标志 → 读采样 → 算法 → 写执行器 → 可选 `BM_SNAPSHOT_PUBLISH`。**禁止**打日志、分配内存、处理事件订阅。

## 4. Scheduled HRT 用法（`bm_hrt`）

用于**不需要与 PWM 相位锁定**的中低频硬实时任务：速度环 1 kHz、BMS 积分 100 Hz 等。

### 4.1 直接使用 `bm_hrt`

```c
static void speed_loop(void *ctx) {
    axis_t *a = (axis_t *)ctx;
    a->velocity_ref = ...;
}

static const bm_hrt_slot_t slots[] = {
    {
        .period_us = 1000,
        .trigger   = BM_HRT_TRIGGER_TIMER,
        .callback  = speed_loop,
        .context   = &axis,
        .name      = "speed",
    },
};
bm_hrt_init(slots, 1);
bm_hrt_start();
```

约束：

- `period_us` 须为 `BM_CONFIG_HRT_TICK_US` 的整数倍（默认 100 µs → 1 ms 合法）。
- 槽数 ≤ `BM_CONFIG_HRT_MAX_SLOTS`。
- 错过 deadline 时 `bm_hrt_deadline_missed_hook` 弱符号可被覆盖。

### 4.2 通过 `bm_ctrl_inst`（多实例自动组装）

声明 `BM_CTRL_SLOT_SCHEDULED` 后，`bm_ctrl_init_all()` 把所有实例的 Scheduled 槽合并为一张 `bm_hrt` 表，**只调用一次** `bm_hrt_init()`。

应用仍需在 `init_all` 之后、`start_all` 之前（或按示例顺序）调用 `bm_hrt_start()` 启动定时器 ISR。

## 5. SRT 用法

SRT 域运行在主循环，包含两套互补机制：

| 机制 | 作用 | 典型场景 |
|------|------|----------|
| `bm_event` | 异步发布/订阅，带优先级队列 | 模块解耦、ISR 通知主循环 |
| `bm_ticker` | 毫秒周期，到期后 **发布空事件** | 把周期任务转为事件驱动 |
| `bm_module` | init/start/stop 生命周期 | **开机顺序**，不是运行期消息 |
| `bm_channel` | SPSC 环形缓冲 | SRT 内生产者/消费者 |

### 5.1 `bm_module` vs `bm_event`

- **`bm_module`**：管「上电」——按 `priority` 升序 `init` / `start`；失败回滚。
- **`bm_event`**：管「运行中」——发布方入队，主循环 `bm_event_process()` 出队调订阅者。

二者常一起用，但职责不同。参见 [07-应用骨架与数据流](07-应用骨架与数据流.md)。

### 5.2 ISR 向 SRT 发信号

`interrupt_demo` 演示：ISR 只 `bm_event_publish_copy_from_isr()`，**订阅回调仍在主循环执行**。

适用：**低频通知**（按键、故障、统计），不适用高频数据流。

## 6. 同步域 `bm_sync`

### 6.1 解决什么问题

同步域解决 **多路 PWM/ADC 的硬件时序对齐**，例如：

- 多关节机器人：各轴同时起步；
- 交错并联电源：各相固定相位差；
- 多 ADC 注入序列在同一 master 相位触发。

**同步域不负责数据通讯**。电流、位置、电压等仍走 `bm_snapshot` 或事件。

### 6.2 描述符

```c
typedef struct {
    const char *name;
    const bm_hal_timer_t *master_timer;   /* 主定时器（TRGO 等） */
    const bm_ctrl_inst_t *const *members; /* 参与同步的实例 */
    const uint32_t *phase_ticks;          /* 各成员相对主相位的 tick 偏移 */
    uint32_t member_count;
} bm_sync_domain_t;
```

### 6.3 状态机

```text
configure ──► HAL 配置主从定时器连线，不输出
    │
    ▼
  arm ──────► 武装，仍不启动 PWM
    │
    ▼
 trigger ───► 按 phase_ticks 同时启动各成员
    │
    ▼
safe_stop ──► 安全停机，清框架层与 HAL 状态
```

框架层（`bm_sync.c`）校验描述符、维护 `g_active_domain` / `g_armed`，并委托 `bm_sync_hal_*` 完成硬件操作。

### 6.4 典型调用顺序

与 `examples/multi_axis_sync/main.c` 一致：

```c
bm_ctrl_init_all(instances, count);
bm_sync_configure(&domain);
bm_sync_arm(&domain);
bm_hrt_start();
bm_ctrl_start_all(instances, count);
bm_sync_trigger(&domain);   /* 多轴同 tick 进入 step */
```

切换同步域时，`bm_sync_configure()` 会对旧域调用 `safe_stop`。

### 6.5 HAL 链接注意

未链接平台 HAL 时，`bm_sync_hal_stub.c` 返回 `BM_ERR_NOT_INIT`。PC 仿真须链接 `hal_reference/native_sim/bm_sync_hal_native.c`；真机见 `hal_reference/stm32g4/`。

## 7. 跨域通讯

### 7.1 选型矩阵

| 方向 | 传什么 | 推荐机制 | 不推荐 |
|------|--------|----------|--------|
| HRT → SRT | 最新测量值、状态字 | `BM_SNAPSHOT_PUBLISH` / `READ` | `bm_event` 高频捎带 |
| HRT(快) → HRT(慢) | 降采样结果 | 同上 `bm_snapshot` | 共享无保护全局变量 |
| ISR → SRT | 「发生了」+ 小 payload | `bm_event_publish_copy_from_isr` | 在 ISR 里调订阅者 |
| SRT → SRT | 业务消息 | `bm_event` / `bm_channel` | — |
| SRT → HRT | 设定值、使能位 | 反向 snapshot 或单写者 `volatile` 字段 | SRT 持锁写复杂结构 |

**原则**：事件是 **信号**（通知刷新），不是 **数据管道**。实时流走 snapshot。

### 7.2 `bm_snapshot` 三缓冲原理

头文件组件 `include/bm_snapshot.h`，无 `.c` 实现。

```c
BM_SNAPSHOT_DEFINE(pack_current, float);
static pack_current_snapshot_t g_pack = BM_SNAPSHOT_INITIALIZER;
```

**写者**（HRT，如 ADC 完成）：

```c
float amps = ...;
BM_SNAPSHOT_PUBLISH(g_pack, &amps);
```

**读者**（SRT 或慢 HRT）：

```c
float amps;
BM_SNAPSHOT_READ(g_pack, &amps);
```

内部逻辑：

1. 三份 `buffer[3]`，维护 `published`（最新完成帧）与 `reading`（读者占用）。
2. 写者选「既不是 published 也不是 reading」的槽写入，内存屏障后 flip `published`。
3. 读者标记 `reading`，拷贝，再释放。若写者在此期间再次发布，读者可能重试（`do-while` 保证一致性拷贝）。

特性：

- 无锁，适合单写者单读者；
- 允许丢中间帧，读者始终拿到 **最新完成的一帧**；
- 适合传感器、电流、位置等「最新值即可」的量。

### 7.3 示例对照

| 示例 | 跨域路径 |
|------|----------|
| `hrt_bms_coulomb` | Pack Hardware HRT `PUBLISH` 电流 → 电芯 Scheduled HRT `READ` 积分 |
| `multi_channel_bms` | Pack Hardware HRT 写 16 路电压 snapshot → SRT ticker 事件触发后批量 `READ` |
| `hrt_servo_stub` | 三环在同一 `servo_state_t`；位置通知用 SRT 事件（非 snapshot 演示） |
| `interrupt_demo` | GPIO ISR → `publish_copy_from_isr` → 主循环处理 |

### 7.4 SRT → HRT 反向路径

文档与示例较少涉及，常见模式：

1. **反向 snapshot**：SRT `PUBLISH` 设定值，HRT `READ`（注意 WCET：HRT 读路径须极简）。
2. **单写者约定**：SRT 写 `volatile` 标量或对齐的结构，HRT 只读；启用前用 `bm_memory_barrier_release()`。
3. **使能/模式**：通过 `bm_ctrl_inst` 的 `ops->start` / `safe_stop` 在 SRT 侧切换，而非在环内改复杂状态机。

设计时须明确 **谁写、谁读、是否允许丢帧**。

## 8. `bm_ctrl_inst` 如何把两条 HRT 路径焊在一起

`bm_ctrl_init_all()` 内部流程（简化）：

```text
ctrl_teardown_session()          /* 重入时先清理 */
  → validate_instance / 资源冲突检查
  → assemble_tables()
        每个 slot 记入 g_bindings[]
        SCHEDULED 槽 → 填入 g_hrt_slots[]
  → bm_hrt_init(g_hrt_slots, ...)   /* 仅有 Scheduled 槽时 */
  → 各实例 ops->init()
  → 各 HARDWARE 槽 bind_hardware(hal_binding)
```

`bm_ctrl_start_all()` 只调各实例 `ops->start()`（如使能 PWM），**不**启动 `bm_hrt`——须由应用调 `bm_hrt_start()`。

`bm_ctrl_safe_stop_all()` / 初始化失败：停 HRT → safe_stop → 解绑硬件 → `bm_hrt_reset()`。

## 9. 伺服三环数据流（综合示例）

`hrt_servo_stub` 分层：

```text
ADC IRQ ──► current_step (Hardware HRT)     改 duty
TIM tick ─► speed_step (Scheduled HRT)    改 position 计数
主循环 ───► bm_ticker_poll → EVENT_POSITION → on_position_event
```

若 UI 要显示 position：**不要**只靠事件里的空 payload；应 `BM_SNAPSHOT_PUBLISH` position，或 SRT 在事件回调里读 `instance->state`（须保证 HRT 只写、SRT 只读的字段语义）。

## 10. CMake 与配置速查

| 需求 | CMake 选项 |
|------|------------|
| Scheduled HRT | `BM_ENABLE_HRT` |
| SRT 周期 ticker | `BM_ENABLE_TICKER` |
| 多实例控制 | `BM_ENABLE_CTRL_INST`（依赖 HRT） |
| 同步域 | `BM_ENABLE_SYNC`（依赖 CTRL_INST） |

`bm_config.h` 关键项：

- `BM_CONFIG_HRT_TICK_US`：Scheduled HRT 时间粒度；
- `BM_CONFIG_HRT_MAX_SLOTS`：Scheduled 槽上限；
- `BM_CONFIG_MAX_CTRL_SLOTS` / `BM_CONFIG_MAX_CTRL_INSTANCES`：控制实例规模。

详见 [05-构建配置与CMake](05-构建配置与CMake.md)。
