# 09 测试与调试

## 测试金字塔

| 层级 | 目标 | 环境 |
|------|------|------|
| 单元测试 | 算法、状态机、边界条件 | PC + `bm_hal_native` |
| QEMU 冒烟 | 启动、中断上下文、HAL 集成 | `qemu_cortex_m0` 等 |
| 硬件在环 | 时序、功耗、外设 | 参考板 + 脚本 |

## 单元测试

```bash
cmake -B build -DBM_BUILD_TESTS=ON \
  -DBM_ENABLE_CHANNEL=ON -DBM_ENABLE_SHELL=ON \
  -DBM_ENABLE_HRT=ON -DBM_ENABLE_TICKER=ON \
  -DBM_ENABLE_CTRL_INST=ON -DBM_ENABLE_SYNC=ON \
  -DBM_SYNC_HAL_NATIVE=ON
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

框架：`tests/unit/`，Unity + fff mock。每个组件有对应 `test_*.c`。

### 编写测试约定

- `setUp()` 中调用 `bm_event_reset()`、`bm_wdg_reset()`、`bm_module_deinit_all()` 等，保证用例隔离。
- 混合域测试可能依赖 `bm_hal_timer_native` 推进时间。
- 不依赖堆；静态分配与生产代码一致。

## QEMU 冒烟

```bash
cd examples
./run_qemu.sh interrupt_demo   # 或 run_qemu.ps1
```

`tests/qemu/` 含架构级冒烟用例（若已启用 CI）。

## 调试手段

| 手段 | 适用 |
|------|------|
| `bm_log` + UART | SRT 诊断；可重定向到 RTT/SWO |
| `bm_shell` | 运行时查询/注入命令（**勿驱动安全动作**） |
| 断言与错误码 | 检查 `BM_ERR_*` 返回值，fail-stop 保留现场 |
| `bm_event_get_dropped_count()` | 队列溢出监控 |

**禁止**在 Hardware/Scheduled HRT ISR 中打日志或跑 Shell。
