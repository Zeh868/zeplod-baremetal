# 00 快速开始

本文帮助你在 **10 分钟内**完成：拉取代码 → 构建 → 运行示例或单元测试。无需真实硬件。

## 环境要求

| 场景 | 工具 |
|------|------|
| PC 本地模拟 | CMake 3.20+、C 编译器（MSVC / GCC / Clang） |
| 单元测试 | 同上 |
| QEMU 示例 | `arm-none-eabi-gcc`、`qemu-system-arm` |
| Windows 构建 | 推荐 Visual Studio 生成器或 Ninja |

## 1. 克隆与目录

```bash
git clone <repo-url> zeplod-baremetal
cd zeplod-baremetal
```

框架源码在 `include/`、`src/`；平台 HAL 参考在 `hal_reference/`；示例在 `examples/`。

## 2. 运行第一个示例（PC 模拟）

以 `core_sensor` 为例，演示事件、内存池与模块生命周期：

```bash
cmake -B build -S examples/core_sensor -DZEPLOD_BAREMETAL_DIR=../..
cmake --build build
./build/core_sensor          # Linux/macOS
build\Debug\core_sensor.exe  # Windows MSVC
```

成功时终端会打印模块初始化与温度事件处理日志。

**PowerShell 快捷脚本**（在 `examples/` 目录）：

```powershell
.\run_single.ps1 core_sensor
```

## 3. 运行单元测试

```bash
cmake -B build -DBM_BUILD_TESTS=ON \
  -DBM_ENABLE_CHANNEL=ON \
  -DBM_ENABLE_SHELL=ON \
  -DBM_ENABLE_HRT=ON \
  -DBM_ENABLE_TICKER=ON \
  -DBM_ENABLE_CTRL_INST=ON \
  -DBM_ENABLE_SYNC=ON
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

全部通过即表示核心与混合域组件在 `native_sim` HAL 上行为正常。

## 4. QEMU 示例（可选）

```bash
cmake -B build_qemu -S examples/interrupt_demo \
  -DZEPLOD_BAREMETAL_DIR=../.. \
  -DBOARD=qemu_cortex_m0
cmake --build build_qemu
qemu-system-arm -M microbit -kernel build_qemu/interrupt_demo.elf -nographic -serial stdio
```

## 5. 创建自己的应用

最小步骤：

1. 复制 `examples/core_sensor/` 为模板。
2. 编写应用侧 `bm_config.h`（队列深度、模块数等上限）。
3. 在 `CMakeLists.txt` 中 `target_link_libraries` 链接所需目标（如 `bm_core`、`bm_module`）。
4. 链接平台 HAL：`bm_hal_native`（PC）或 `bm_hal_stm32g4` 等。
5. 在 `main()` 中：`bm_event_reset()` → `bm_module_init_all()` → 主循环 `bm_event_process()`。

详见 [05-构建配置与CMake](05-构建配置与CMake.md)、[06-示例与上手路径](06-示例与上手路径.md) 与 [07-应用骨架与数据流](07-应用骨架与数据流.md)。
