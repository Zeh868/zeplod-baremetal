# 00 快速开始

> **本文职责**：10 分钟内完成拉取代码 → 构建 → 运行示例或单元测试（无需真实硬件）。

本文帮助你在 **10 分钟内**完成：拉取代码 → 构建 → 运行示例或单元测试。无需真实硬件。

## 环境要求

| 场景 | 工具 |
|------|------|
| PC 本地模拟 | CMake 3.20+、C 编译器（MSVC / GCC / Clang） |
| 单元测试 | 同上 |
| QEMU 示例 | `arm-none-eabi-gcc`、`qemu-system-arm` |
| Windows 构建 | 推荐 Visual Studio 生成器或 Ninja |

## 1. 克隆与目录

```bash
git clone <repo-url> zeplod-baremetal
cd zeplod-baremetal
```

框架源码在 `Source/`、`include/`；平台 Port 在 `portable/`；示例在 `Demo/`。

## 2. 运行第一个示例（PC 模拟）

以 `core_sensor` 为例，演示事件、内存池与模块生命周期：

```bash
cmake -B build/demo/manual/core_sensor -S Demo/core_sensor -DZEPLOD_BAREMETAL_DIR=../..
cmake --build build/demo/manual/core_sensor
./build/demo/manual/core_sensor/core_sensor          # Linux/macOS
build\demo\manual\core_sensor\Debug\core_sensor.exe  # Windows MSVC
```

成功时终端会打印模块初始化与温度事件处理日志。

**PowerShell 快捷脚本**：

```powershell
.\tools\demo\run_native.ps1 core_sensor
```

## 3. 运行单元测试

```bash
cmake -B build -DBM_BUILD_TESTS=ON \
  -DBM_ENABLE_CHANNEL=ON \
  -DBM_ENABLE_SHELL=ON \
  -DBM_ENABLE_HRT=ON \
  -DBM_ENABLE_TICKER=ON \
  -DBM_ENABLE_CTRL_INST=ON \
  -DBM_ENABLE_SYNC=ON
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

全部通过即表示核心与混合域组件在 `native_sim` HAL 上行为正常。

## 4. QEMU 示例（可选）

```bash
.\tools\demo\run_qemu.ps1 interrupt_demo
# 或手动：
# cmake -B build/demo/qemu/interrupt_demo -S Demo/interrupt_demo ...
```

## 5. 创建自己的应用

最小步骤：

1. 复制 `Demo/core_sensor/` 为模板。
2. 编写应用侧 `bm_config.h`（容量上限 + `BM_CONFIG_ENABLE_*` 组件开关）。
3. Include Path：`你的工程/`（`bm_config.h`）+ 框架 `include/`（**一条**）。
4. 应用代码：`#include "zeplod.h"`；板级 HAL 按需 `#include "bm_hal_*.h"`。
5. CMake：`zeplod_configure` + `zeplod_link`，或手动链接 `bm_core`、`bm_module` 等目标与 `bm_hal_*` 后端。
6. 在 `main()` 中：`bm_event_reset()` → `bm_module_init_all()` → 主循环 `bm_event_process()`。

详见 [05-构建配置与CMake](05-构建配置与CMake.md)、[06-示例与上手路径](06-示例与上手路径.md) 与 [07-应用骨架与数据流](07-应用骨架与数据流.md)。
