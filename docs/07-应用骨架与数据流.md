# 07 应用骨架与数据流

本文回答：**从空白工程到跑起来，应用代码长什么样、初始化顺序是什么、数据怎么流**。配合 [06-示例与上手路径](06-示例与上手路径.md) 中的示例列表使用。

执行域原理见 [03-执行域与跨域通讯](03-执行域与跨域通讯.md)。

---

## 1. 框架在应用里的位置

```text
┌─────────────────────────────────────────────────────────┐
│  你写的应用                                              │
│  · bm_config.h（容量上限）                               │
│  · 模块表 / 控制实例表 / 事件订阅                        │
│  · main() 初始化 + 主循环                                │
├─────────────────────────────────────────────────────────┤
│  Zeplod 框架（src/core、src/hybrid）                     │
│  · 事件、池、模块、HRT、ticker、ctrl_inst、sync          │
├─────────────────────────────────────────────────────────┤
│  平台 HAL（hal_reference/ 或你的板级实现）               │
│  · UART、定时器、PWM、ADC、临界区                        │
└─────────────────────────────────────────────────────────┘
```

**你写**业务逻辑与表驱动描述；**框架跑**队列、调度、校验与回滚；**HAL 跑**寄存器与中断。

## 2. 按层级选择骨架

| 层级 | 何时用 | 主要 API | 参考示例 |
|------|--------|----------|----------|
| **Nano** | 传感器、简单状态机 | `bm_module` + `bm_event` | `core_sensor` |
| **Nano+** | 外设中断通知主循环 | + `publish_copy_from_isr` | `interrupt_demo` |
| **Lite** | 多模块 + 调试 + 看门狗 | + `bm_channel`、`bm_shell`、`bm_wdg` | `full_system` |
| **Control** | 伺服、BMS、多轴 | + `bm_ctrl_inst`、`bm_hrt`、`bm_snapshot`、`bm_sync` | `hrt_servo_stub` |

CMake 选项与链接目标见 [05-构建配置与CMake](05-构建配置与CMake.md)。

## 3. Nano 骨架：`core_sensor`

### 3.1 应用要准备的四件事

| 文件/代码 | 内容 |
|-----------|------|
| `bm_config.h` | `BM_CONFIG_EVENT_QUEUE_SIZE`、`BM_CONFIG_MAX_MODULES` 等 |
| 事件类型 | `bm_event_register_type(EVENT_TEMP, "TEMP")` |
| 模块表 | `_bm_module_table[]`：sensor / display / logger 的 init、start |
| `main()` | 初始化 → 主循环 `bm_event_process()` |

### 3.2 初始化顺序

```c
int main(void) {
    bm_hal_uart_init(NULL);          /* 1. 平台 HAL */
    bm_event_reset();                /* 2. 事件系统复位 */
    bm_event_register_type(...);     /* 3. 注册类型 */
    bm_event_subscribe(...);         /* 4. 订阅 */
    bm_module_init_all();            /* 5. 模块 init（按 priority） */
    bm_module_start_all();           /* 6. 模块 start */
    for (;;) { ... }                 /* 7. 主循环 */
}
```

### 3.3 一周期数据流

```text
sensor 模块 / 主循环
    │
    │  bm_mempool_alloc → 填数据
    ▼
bm_event_publish_copy(EVENT_TEMP, prio, &data, sizeof(data))
    │
    │  入队（异步，此时订阅者未执行）
    ▼
bm_mempool_free
    │
    ▼
bm_event_process()
    │
    ├──► display_on_temp()   /* 订阅者 1 */
    └──► logger_on_temp()    /* 订阅者 2 */
```

要点：`publish` **只入队**；回调在 `bm_event_process` 里执行。这与「直接调函数」的裸机写法不同，是 Nano 层的核心心智模型。

### 3.4 最小 CMake

```bash
cmake -B build -S examples/core_sensor -DZEPLOD_BAREMETAL_DIR=../..
cmake --build build
```

示例 `CMakeLists.txt` 已链接 `bm_core`、`bm_module`、平台 HAL，无需手写 `src/core/*.c` 列表。

## 4. Lite 骨架：`full_system`

在 Nano 基础上增加：

```c
bm_shell_init(&shell);
bm_wdg_register("comm");
bm_wdg_register("control");

for (;;) {
    publish_or_poll_sensors();
    bm_event_process();
    bm_shell_poll(&shell);
    bm_wdg_feed_module("comm");
    bm_wdg_feed_module("control");
    bm_wdg_feed();               /* 全部 feed 后才喂硬件狗 */
}
```

- **`bm_channel`**：仅 SRT 内单生产者单消费者；禁止 HRT ISR 访问。
- **`bm_module`** 仍只管启动；运行期协作靠 `bm_event`。

## 5. Control 骨架：混合域

### 5.1 控制实例描述

每个算法实例（一轴伺服、一个 Pack 采样器）用 `bm_ctrl_inst_t` 描述：

```c
typedef struct {
    uint32_t id;
    const char *name;
    void *state;                              /* 实例私有状态 */
    const void *config;                       /* 硬件句柄等 */
    const bm_ctrl_slot_t *slots;              /* 执行槽 */
    uint32_t slot_count;
    const bm_resource_claim_t *claims;        /* 资源占用声明 */
    const bm_ctrl_ops_t *ops;                 /* init / start / safe_stop */
} bm_ctrl_inst_t;
```

每个 **slot** 二选一：

| kind | 含义 | 你需要提供 |
|------|------|------------|
| `BM_CTRL_SLOT_HARDWARE` | Hardware HRT | `step` + `bind_hardware` |
| `BM_CTRL_SLOT_SCHEDULED` | Scheduled HRT | `step` + `period_us` |

### 5.2 正确初始化顺序

```c
static const bm_ctrl_inst_t *const g_instances[] = { &axis0, &axis1 };

int main(void) {
    bm_hal_uart_init(NULL);
    bm_event_reset();                    /* 若用 ticker/事件 */

    /* 可选：定时器基准（ticker 与 HRT 可能共用 HAL timer） */
    (void)bm_hal_timer_init(10000u);

    /* ① 校验、组 HRT 表、绑 Hardware、各实例 init */
    rc = bm_ctrl_init_all(g_instances, COUNT);

    /* ② 可选：同步域（多轴） */
    rc = bm_sync_configure(&domain);
    rc = bm_sync_arm(&domain);

    /* ③ 启动 Scheduled HRT 定时器 ISR */
    rc = bm_hrt_start();

    /* ④ 各实例 start（使能 PWM 等） */
    rc = bm_ctrl_start_all(g_instances, COUNT);

    /* ⑤ 可选：同步触发 */
    rc = bm_sync_trigger(&domain);

    /* ⑥ 可选：SRT ticker */
    rc = bm_ticker_init(ticker_slots, ticker_count);

    for (;;) {
        bm_ticker_poll();
        bm_event_process();
        bm_wdg_feed();
    }
}
```

**常见错误**：

| 错误 | 后果 |
|------|------|
| 不调 `bm_hrt_start()` | Scheduled 槽永不执行 |
| `start_all` 在 `hrt_start` 之前且依赖 HRT tick | 时序错乱 |
| `init_all` 失败未处理 | 半初始化状态 |
| 在 Hardware ISR 里 `BM_LOG` | WCET 膨胀、可能死锁 |

`hrt_servo_stub` 与 `multi_axis_sync` 均遵循上述顺序（多轴示例在 `start_all` 后 `sync_trigger`）。

### 5.3 运行时两条路径

```text
时间 ─────────────────────────────────────────────────────►

ISR:  |--ADC--|     |--ADC--|     |--ADC--|   Hardware HRT
ISR:  |---1ms---|   |---1ms---|   |---1ms---|  Scheduled HRT

主循环:  [ticker][event][wdg]  [ticker][event][wdg] ...     SRT
```

主循环 **不** 调用 HRT 回调；HRT 在独立 ISR 链上运行。

## 6. 示例数据流对照

### 6.1 `hrt_servo_stub` — 三环分工

| 环 | 域 | 槽 | 做什么 |
|----|-----|-----|--------|
| 电流 | Hardware HRT | ADC 完成 | 读采样、写 PWM duty |
| 速度 | Scheduled HRT | 1 ms | `position++` |
| 位置通知 | SRT | ticker → 事件 | `on_position_event` 计数 |

跨域：位置 **数值**在 `g_servo_state`；事件只表示「该刷新了」。生产代码建议对 UI 暴露的量使用 `bm_snapshot`。

### 6.2 `hrt_bms_coulomb` — snapshot 跨 HRT 速率

```text
Pack ADC 完成 (Hardware HRT)
    │  BM_SNAPSHOT_PUBLISH(pack_current)
    ▼
电芯积分 (Scheduled HRT, 如 1 ms)
    │  BM_SNAPSHOT_READ → coulomb_ah +=
    ▼
比较器过流 (Hardware) → safe_stop
```

### 6.3 `multi_channel_bms` — 多路 snapshot + SRT 检测

```text
Pack 采样 (Hardware HRT)
    │  循环 16 通道 BM_SNAPSHOT_PUBLISH(cell_voltage[i])
    ▼
ticker 事件 (SRT)
    │  bm_event_process → 订阅者 BM_SNAPSHOT_READ 各通道
    ▼
过压判断 / 上报
```

### 6.4 `multi_axis_sync` — 同步域无时序数据

```text
sync_trigger
    │  三轴同一 HRT tick 进入 axis_step
    ▼
各轴 start_tick 相同 → 验证同步成功
```

同步域不传递测量值；只保证 **同时起步**。

### 6.5 `interrupt_demo` — ISR 只发信号

```text
SysTick / 外设 ISR
    │  bm_event_publish_copy_from_isr(...)
    ▼
主循环 bm_event_process → 订阅者执行业务
```

## 7. 从 Nano 升级到 Control 要加什么

| 步骤 | 动作 |
|------|------|
| 1 | `bm_config.h` 增加 `BM_CONFIG_HRT_*`、`BM_CONFIG_MAX_CTRL_*` |
| 2 | CMake 打开 `BM_ENABLE_HRT`、`BM_ENABLE_CTRL_INST`（多轴再加 `BM_ENABLE_SYNC`） |
| 3 | 把环算法从 `main` / 模块里拆成 `bm_ctrl_slot_t` 的 `step` |
| 4 | Hardware 环写 `bind_hardware`；Scheduled 环写 `period_us` |
| 5 | 用 `bm_ctrl_init_all` 替换手写 `bm_hrt_init`（多实例时） |
| 6 | HRT→SRT 数据改 `bm_snapshot`；ISR 通知保留 `publish_copy_from_isr` |
| 7 | 主循环保留 `bm_event_process`；周期任务可迁到 `bm_ticker` |

## 8. 构建与目录约定

```text
your_app/
├── bm_config.h          /* 容量与 HRT tick */
├── main.c
├── CMakeLists.txt       /* target_link_libraries(... bm_core bm_hrt ...) */
└── board/               /* 可选：你的 HAL 实现 */
```

框架源码布局（仓库内）：

```text
src/core/     event, mempool, module, channel, shell, wdg, ...
src/hybrid/   hrt, ticker, ctrl_inst, resource, sync
src/hal/      弱符号桩（未链平台 HAL 时）
hal_reference/  native_sim, stm32g4, qemu_cortex_m0, ...
```

应用 **不** 应复制 `src/` 文件；通过 CMake 链接 `bm_core`、`bm_hrt` 等目标即可。

## 9. 调试检查清单

| 现象 | 检查 |
|------|------|
| Scheduled 环从不执行 | 是否 `bm_hrt_start()`？`period_us` 是否整除 `BM_CONFIG_HRT_TICK_US`？ |
| Hardware 环不执行 | `bind_hardware` 是否成功？HAL 是否触发 IRQ/仿真 fire？ |
| 事件订阅不触发 | 是否 `bm_event_process()`？类型是否 `register`？ |
| 多轴不同步 | 是否 `sync_configure` → `arm` → `hrt_start` → `start_all` → `trigger`？HAL 是否链接？ |
| 数据乱跳 | HRT/SRT 是否共写同一结构而无 snapshot？ |
| 初始化失败 | 查日志 `ctrl` / `resource`；资源 claim 是否冲突？ |

日志与单元测试见 [09-测试与调试](09-测试与调试.md)。
