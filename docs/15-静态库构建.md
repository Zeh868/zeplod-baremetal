# 15 静态库构建

> **本文职责**：用 CMake 预编译 `libbm_*.a` 并在应用中链接。  
> **不负责**：Port 实现 → [14-Port移植层](14-Port移植层.md)。

## 1. 前提

静态库只包含 `Source/` 内核；**应用工程仍须编译 Port**（`portable/template/bm_port.c` 的副本）。

集成总览见 [13-集成到现有工程](13-集成到现有工程.md)。

## 2. 构建命令

脚本入口：[`cmake/static-lib/CMakeLists.txt`](../cmake/static-lib/CMakeLists.txt)

```bash
cmake -B build -S cmake/static-lib \
    -DCMAKE_TOOLCHAIN_FILE=cmake/toolchain-arm-none-eabi.cmake \
    -DZEPLOD_PROFILE=event
cmake --build build
```

## 3. 应用侧链接

1. 将生成的 `libbm_*.a` 与 `include/` 加入工程。
2. 编译应用自己的 `bm_port.c`。
3. 提供应用目录下的 `bm_config.h`（置于 include 路径首位）。

Keil / IAR 链接 `.a` 的步骤见 [18-Keil集成](18-Keil集成.md)、[19-IAR集成](19-IAR集成.md)。
