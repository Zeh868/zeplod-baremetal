# 11 安全与可靠性

> **本文职责**：fail-stop 策略、有界执行与安全需求追溯摘录。

本文说明框架在**功能安全准备**方面的行为边界，**不代表**已通过 IEC 61508 / SIL 正式认证。

> 状态：认证准备工件（非正式 SIL-2 证书）  
> 范围：`src/` 核心与可选组件  
> 故障策略：**fail-stop**（返回确定性错误码，不自动复位控制块）

## 故障策略：fail-stop

运行时发现以下情况时，返回确定性错误码（如 `BM_ERR_INVALID`、`BM_ERR_OVERFLOW`、`BM_ERR_NOT_INIT`），**不自动复位**控制块或掩盖故障：

- 索引越界、算术溢出
- 非法状态机转换
- 重复释放、双注册
- 启动校验失败

由应用或上层监督逻辑决定：重试、降级、安全停机。

框架在单 MCU 上采用**每子系统单静态运行时**（非多进程式多实例）。设计动机、子系统边界与 `bm_*_get_session` / `bm_*_is_started` 等只读查询 API 见 [12-运行时与实例模型](12-运行时与实例模型.md)。

## 安全目标

1. 运行时检测无效控制块、索引越界、算术溢出，返回 `BM_ERR_INVALID` / `BM_ERR_OVERFLOW`。
2. SRT 组件（`bm_event`、`bm_mempool`、`bm_channel`）在临界区内保持索引一致性。
3. 启动/停机失败进入可诊断状态；`bm_ctrl_start_all` 失败时停止 HRT 并解绑硬件。
4. 软件看门狗在 `BM_CONFIG_WDG_MODULE_TIMEOUT_MS` 窗口内要求各模块 `feed_module`。

## 域隔离与设计规则

| 规则 | 原因 |
|------|------|
| HRT 不访问 `bm_event` / `bm_channel` / `bm_mempool` | 避免 WCET 不可控与死锁 |
| `bm_shell` 仅调试构建 | 防未授权命令影响安全输出 |
| 启动前资源冲突检测 | `bm_ctrl_init_all` 校验声明 |
| `start_all` 失败回滚 | 停 HRT、解绑、已启动实例逆序停止 |
| 软件看门狗模块窗口 | 未 `feed_module` 则不喂硬件狗 |

## 残余风险（使用前必读）

1. **HAL 弱符号**：未链接平台 HAL 时，外设调用可能静默失败 → 生产构建必须链接 `bm_hal_<platform>`。
2. **零拷贝事件**：`bm_event_publish_event` 生命周期由调用方保证。
3. **Shell 缓冲区**：`exec` 就地修改 `line`；命令处理函数若异步持有 `argv` 指针需注意缓冲区生命周期。
4. **WCET**：已将内存池清零、看门狗字符串/HAL/时间换算和事件日志移出临界区；最终上界仍需结合目标 MCU、编译选项与实测填写。
5. **事件回调非重入**：`bm_event_process` 回调内的 subscribe/unsubscribe/常规 publish/process 会被运行时拒绝并计数；SRT ISR 使用 `*_from_isr` 发布。`bm_ultra_process` 仍要求回调不重入。
6. **`bm_ultra_queue_state`**：无锁只读，禁止与队列并发访问。
7. **测试注入钩子**：`bm_event_test_inject` / `bm_ultra_test_inject` 仅在定义 `BM_ENABLE_EVENT_TEST_HOOK` / `BM_ENABLE_ULTRA_TEST_HOOK` 时编译进 `bm_core`；生产固件不得定义上述宏。
8. **零模块看门狗**：未注册任何模块时 `bm_wdg_feed` 直接喂硬件狗；安全应用应在启动阶段至少注册监督模块。
9. **`bm_mempool_reset`**：仅复位位图，不清理池内对象；调用方须保证无悬空指针。

## 医疗 / 汽车高安全等级

框架提供**可审计、可测试**的编码风格与文档追踪，**不**替代：

- 危害分析与 FMEA 流程
- 独立第三方 SIL/ASIL 认证
- 冗余硬件与诊断覆盖率目标

不适用场景见 [README.zh-CN.md](../README.zh-CN.md)「覆盖领域」章节。

## 验证命令

```bash
cmake -B build-sil2 -DBM_ENABLE_CHANNEL=ON -DBM_ENABLE_SHELL=OFF \
  -DBM_ENABLE_HRT=ON -DBM_ENABLE_TICKER=ON -DBM_ENABLE_CTRL_INST=ON \
  -DBM_ENABLE_SYNC=ON -DBM_SYNC_HAL_NATIVE=ON
cmake --build build-sil2 --config Debug
ctest --test-dir build-sil2 -C Debug --output-on-failure
```

更多测试说明见 [09-测试与调试](09-测试与调试.md)。

## 建议的工程实践

1. 生产固件关闭 `BM_ENABLE_SHELL` 或剥离命令表。
2. CI 跑全量单元测试 + QEMU 冒烟。
3. 对安全相关路径维护需求—测试双向追溯（上表为起点）。
4. 关键回调避免动态行为；HRT 路径保持无分支深调用。
